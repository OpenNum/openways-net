# 03-五文件映射建议

> 目标：将“开锁服务 Locksmith”行业资料映射成一个可执行的 AI 行动包，重点解决紧急服务接待、报价预估、身份验证、住宅/商业/汽车分诊、客服与调度协同。

---

## 一、整体评估

### 行业是否适合做 AI 行动包
适合，而且优先级高。

### 原因
1. 来电和询盘高度标准化，适合 AI 做首轮分诊。
2. 客户最在乎速度、价格区间、是否可信，AI 正好能标准化这三件事。
3. 住宅 / 商业 / 汽车天然可拆成不同流程。
4. 大量询盘发生在下班后、夜间、周末，AI 的 24/7 价值明显。
5. 调度逻辑可规则化，适合与短信、CRM、地图、工单工具连接。

### 风险
- 身份验证不足会带来安全与合规风险。
- 汽车/电子锁业务容易因车型或设备能力不清导致错误承诺。
- 价格若报得过死，现场差异会引发争议。

### 结论
应采用“AI 首轮接待 + 人工复核高风险单 + 技师现场确认最终报价”的混合模式。

## 二、Quick Start Guide

### 1. AI 行动包定位
这是一个面向 locksmith 商家的前台运营 AI，不替代技师，主要负责：
- 接电话/接网页询盘
- 判断是否紧急
- 分类为住宅 / 商业 / 汽车
- 采集身份与授权信息
- 给出价格区间和 ETA 预期
- 派给合适技师或转人工

### 2. 首次上线最小版本
建议先只做 4 个流程：
1. Emergency lockout intake
2. Residential rekey/change lock intake
3. Automotive unlock/key replacement pre-check
4. Commercial callback/quote intake

### 3. 首批必须准备的数据
- 服务区域与 postcode/suburb 覆盖表
- 技师名单、技能标签、值班时间
- 价格区间表
- 车型/品牌能力边界
- 身份验证规则
- 夜间/节假日 surcharge 规则

### 4. 首次上线的目标 KPI
- 漏接率下降
- 首响时间缩短
- 报价前流失率下降
- 派单准确率提升
- 夜间成交率提升

## 三、Tool Guide

### 1. 必需工具

#### A. 电话/网页接待工具
用途：接收电话、网页表单、聊天消息。
要求：
- 支持 24/7 自动接待
- 支持录音/转写
- 支持转人工

#### B. 调度工具
用途：根据区域、技能、值班状态分配工单。
要求：
- 可记录技师当前位置或服务区域
- 可设置优先级
- 可发短信/推送 ETA

#### C. CRM/工单系统
用途：沉淀客户信息、问题类型、报价、结果、复购机会。
关键字段：
- customer_name
- phone
- address
- service_type
- property_type
- urgency
- verification_status
- quoted_range
- assigned_technician
- final_invoice

#### D. 短信通知工具
用途：发送技师姓名、ETA、付款方式、验证要求。

### 2. 推荐数据表结构

#### 服务目录表
- residential_lockout
- residential_rekey
- lock_change
- smart_lock_install
- commercial_lockout
- master_key
- access_control
- auto_unlock
- car_key_replace
- transponder_programming

#### 技师能力表
- technician_name
- suburbs_covered
- emergency_shift
- residential_skill
- commercial_skill
- automotive_skill
- smart_lock_skill
- access_control_skill
- brands_supported

#### 身份验证规则表
- scenario
- required_docs

### 3. 自动化连接建议
- 新来电 -> AI 采集信息 -> 创建工单

- 发送短信给客户：报价区间、ETA、技师信息
- 发送工单给技师：地址、场景、验证状态、备注
- 完工后自动触发评价/复购提醒

## 四、Prompt Library

### Prompt 1：紧急分诊
你是 locksmith 服务前台。你的任务是先判断来电是否为紧急 lockout，并快速收集：服务地址、服务类型（住宅/商业/汽车）、当前是否存在老人/儿童/安全风险、客户联系电话、可否出示身份证明。不要直接承诺固定价格，只给报价区间。

### Prompt 2：住宅场景
你正在处理住宅 locksmith 询盘。先区分是“锁在门外”“换锁”“rekey”“修锁”“智能锁安装”。然后采集门锁类型、地址、时间要求、是否可现场出示照片证件或地址证明，再给区间报价与 ETA。

### Prompt 3：商业场景
你正在处理商业 locksmith 询盘。请确认公司名称、现场地址、授权联系人、问题类型（lockout/master key/access control/panic bar/repair）、是否影响营业、是否需要发票或长期维护报价。高价值或高风险场景自动转人工。

### Prompt 4：汽车场景
你正在处理汽车 locksmith 询盘。请先确认品牌、车型、年份、车牌、钥匙状态（锁车内/丢失/损坏/遥控失灵/需编程）、当前位置、是否能提供驾照与注册/保险信息。若车型或品牌不在支持清单，禁止承诺可做。

### Prompt 5：身份验证
你需要向客户解释：为了保护住户、车主和物业安全，技师到场前或到场时需核验身份与权限。住宅一般需照片证件，汽车一般需驾照与注册/保险，商业需授权联系人确认。若客户无法提供，标记为人工复核。

### Prompt 6：报价说明
请用透明方式报价：先说基础出车费/服务费，再给价格区间，最后说明影响最终价格的因素，如锁型、时段、损坏程度、是否需要材料或编程。不要使用“绝对最低价”“保证最便宜”等词。

## 五、Workflow SOP

### SOP 1：Emergency Lockout
1. 接入来电/聊天
2. 判断是否紧急
3. 识别住宅/商业/汽车
4. 采集地址、电话、场景细节
5. 采集身份验证信息
6. 输出 ETA 和价格区间
7. 匹配技师
8. 发送客户短信与技师工单
9. 到场后现场确认最终价
10. 完工后收款、评价、归档

### SOP 2：Residential Rekey / Lock Change
1. 判断是否为搬家、丢钥匙、租客退租、被盗后处理
2. 确认锁数量和门型
3. 确认是否保留现有五金外观
4. 报工时 + 材料逻辑
5. 派具备住宅与 smart lock 能力的技师

### SOP 3：Commercial / Property Management
1. 确认公司或物业名称
2. 确认授权联系人
3. 判断是紧急恢复营业还是计划性改造
4. 若涉及 access control/master key，自动提升优先级并转人工报价
5. 输出书面报价或回访任务

### SOP 4：Automotive
1. 先核验品牌、车型、年份
2. 判断是 unlock、duplicate、all keys lost、programming 还是 ignition issue
3. 核对证件与车辆归属
4. 检查支持能力与库存
5. 给区间报价
6. 派具有对应编程能力的技师

### SOP 5：高风险/人工升级条件
满足任一条件立即转人工：
- 无法说明与物业/车辆关系
- 身份验证缺失或矛盾
- 商业授权不清
- 涉及高端车型/特殊电子锁
- 客户强烈争议价格
- 夜间且位置存在安全风险

## 六、7-Day Plan

### Day 1
梳理服务目录，拆分住宅/商业/汽车主流程。

### Day 2
建立价格区间表、附加费规则、服务区域表。

### Day 3
建立身份验证规则和人工升级条件。

### Day 4
整理技师能力矩阵，补充车型/锁型支持边界。

### Day 5
配置 AI intake prompts，打通工单与短信通知。

### Day 6
用 20-30 个真实场景压测，覆盖深夜、汽车、商业授权、证件不足等案例。

### Day 7
复盘转化率、报价争议、派单准确率，调整 prompt、SOP 和价格话术。
