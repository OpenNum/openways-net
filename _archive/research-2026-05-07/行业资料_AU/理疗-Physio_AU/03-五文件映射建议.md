# 理疗 / Physiotherapy 五文件映射建议（AI 行动包格式）

更新时间：2026-04-10

## 已理解的既有结构
这类行业目录不是单纯做研究归档，而是为了后续快速产出一个可执行的 AI 行动包。

对理疗 / Physio 赛道来说，最有效的行动包，不应该再按“网站页面切法”拆，而应该按 **获客、沟通、转化、交付、7 天启动** 的真实执行链路拆。

## 我对“是否适合做 AI 行动包”的判断
可以，而且很适合。

原因：
- physio 用户问题明确，典型是疼痛、运动损伤、术后康复、保险/报销流程不清
- 这个行业很依赖标准化解释，而 AI 很适合处理 FAQ、初诊前教育、内容脚本、评价回复、转诊前后沟通
- 合规边界明确，反而更适合先做一套可控模板

# AI 行动包 5 文件建议

## 1. Quick Start Guide

### 目标
让诊所 owner / marketer / 前台在 30 分钟内理解：这个 AI 包能先帮什么，不能帮什么，应该先从哪个入口开始用。

### 建议包含
1. 行业定位
   - 面向澳洲本地 physio / physiotherapy clinic
   - 优先服务华人市场或双语市场
2. 最适合的 4 个使用场景
   - 初诊前 FAQ 自动整理
   - 疼痛场景内容生成
   - 报销 / referral 解释文案
   - 社媒与评价回复草稿
3. 3 个不要做的事
   - 不生成医疗诊断
   - 不承诺疗效或恢复周期
   - 不虚构 Medicare / private health / claim 资格
4. 先从哪个模块开始
   - 综合首页文案
   - 肩颈腰背痛页
   - 费用报销 FAQ
5. 快速上手提示词
   - “把这家 physio 的服务整理成首页卖点”
   - “基于澳洲合规语气，写 10 个 back pain FAQ”

### 为什么它最重要
physio 行业经常不是没有内容，而是不知道从哪里开始，Quick Start Guide 可以先把团队拉到同一语言和边界上。

## 2. Tool Guide

### 目标
告诉团队每个工具或模板在什么场景用，避免乱用 AI。

### 建议包含
1. 输入材料清单
   - clinic services
   - practitioner profile
   - fees / rebate policy
   - suburb / target audience
   - booking link / opening hours
2. 输出能力清单
   - 网站首页草稿
   - 服务页草稿
   - FAQ
   - 小红书 / 微信 / Google post 文案
   - Google review reply
   - 前台解释模板
3. 场景到工具映射
   - 做页面，用行业研究 + prompt library
   - 做社媒，用 prompt library + 7-day plan
   - 做前台 SOP，用 workflow SOP
4. 风险提醒
   - 涉及 women’s health、pelvic floor、post-op、NDIS、WorkCover / CTP 时，必须人工复核
   - 所有报销相关内容必须加 eligibility / referral disclaimer

### 为什么它重要
physio 行业容易把 AI 用成“随便写点文案”，Tool Guide 的作用是把 AI 变成受控生产工具。

## 3. Prompt Library

### 目标
沉淀一组可以直接复制使用的高频 prompt。

### 必须覆盖的 prompt 类型
1. 首页定位 prompt
2. pain point 服务页 prompt
   - back pain
   - sports injury
   - post-op rehab
3. 费用报销 / referral FAQ prompt
4. Google Business Profile post prompt
5. 小红书 / 微信教育型内容 prompt
6. review reply prompt
7. 前台电话 / 微信咨询回复 prompt
8. 合规改写 prompt
   - 把夸大承诺改成澳洲合规表达

### 推荐结构
每个 prompt 包含：
- 适用场景
- 输入字段
- 一键可用 prompt
- 输出示例
- 合规提醒

### 为什么它重要
这个行业的高频沟通高度重复，Prompt Library 一旦做对，团队会立刻提升内容出稿速度。

## 4. Workflow SOP

### 目标
把 AI 使用流程写成操作手册，不依赖单个人经验。

### 建议包含的 SOP
1. 新客户建档 SOP
   - 收集 clinic profile、服务、suburb、目标人群、现有网址
2. 网站文案生成 SOP
   - 先首页，再核心服务页，再 FAQ
3. 社媒内容 SOP
   - 每周选 3 个痛点主题，生成双语版本，人工合规检查后发布
4. 前台沟通 SOP
   - 首次咨询、费用询问、报销询问、是否需要 referral
5. 评论与口碑 SOP
   - 正评感谢模板、低评安抚模板、避免涉及隐私和具体病情
6. 合规复核 SOP
   - 检查疗效承诺、头衔、资质、报销资格、红旗症状提示

### 为什么它重要
physio 诊所很依赖前台和 practitioner 配合，SOP 能把“AI 出稿”变成“团队可落地流程”。

## 5. 7-Day Plan

### 目标
帮助一个 physio 诊所 7 天内把 AI 行动包真的跑起来，而不是停留在文件层面。

### 建议的 7 天安排
**Day 1**：整理诊所资料
- 服务、收费、报销政策、suburb、营业时间、语言支持、照片素材

**Day 2**：产出首页草稿
- 明确 3 个主打场景
- 写第一版 value proposition 和 booking CTA

**Day 3**：产出 2 个服务页
- 肩颈腰背痛
- 运动损伤 或 术后康复

**Day 4**：产出 FAQ
- referral
- private health
- EPC / CDM
- WorkCover / CTP
- first visit

**Day 5**：产出社媒内容 5 条
- 教育型 + 本地化 + 双语可选

**Day 6**：前台话术与 review reply 模板
- 电话、微信、表单咨询统一口径

**Day 7**：合规复核与上线
- 检查夸张用语、资质、claim 说明、CTA 是否清晰

### 为什么它重要
很多诊所最大的问题不是没有想法，而是没有一个能在一周内启动的轻量执行计划。

## 整体评估

### 这 5 个文件是否足够覆盖 physio 行业的 AI 启动需求
**是，且比按页面切更合理。**

因为 physio 行业真正需要的不是“多 5 个页面名字”，而是：
- 如何把复杂治疗流程讲简单
- 如何把合规边界守住
- 如何把前台、内容、FAQ、评价回复统一口径
- 如何在 7 天内做出第一版能上线的资产

### 这套 5 文件里最关键的两个
1. **Prompt Library**，决定出稿效率
2. **Workflow SOP**，决定团队能不能稳定执行

### 如果只能先做 3 个文件，优先级建议
1. Quick Start Guide
2. Prompt Library
3. Workflow SOP

### 当前结论
- **可映射到 AI 行动包 5 文件：是**
- **适合先落地的行业：是**
- **最值得优先做的切入口：首页 + 疼痛场景 + FAQ + 前台沟通模板**
- **整体建议：先做轻量版 AI 行动包，再按 women’s health / post-op / WorkCover 等分支追加专用 prompt**
