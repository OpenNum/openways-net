# 华人医生 / 全科诊所后续五文件映射建议

更新时间：2026-04-10

## 已理解的既有结构
参考现有行业资料包逻辑，这类目录的作用是：
1. 先把行业研究做清楚
2. 再为后续页面 / 品牌 / 社媒生成提供统一依据
3. 让单个行业能独立收口，不和别的行业混在一起

## 我对“是否可映射到 5 文件”的判断
可以，而且很适合。

原因：
- GP 诊所天然适合按高意图就诊场景拆页面
- 用户搜索本来就是按“我现在要看什么问题”进入
- 华人患者对语言、预约、收费和家庭连续照护非常敏感，适合分文件展开

## 更推荐的 5 文件切法

### 方案 A，按“总入口 + 4 个核心业务页”切
这是我最推荐的方式。

1. 首页 / 综合家庭医生入口
   - 讲城市 / suburb、中文沟通、医生团队、预约方式、收费模式、常见服务
2. 家庭全科 / 常见病页
   - 感冒发烧、处方续药、病假条、检查单、转诊、telehealth
3. 儿童与家庭医疗页
   - 儿童常见病、疫苗、湿疹过敏、家庭长期 GP 关系
4. 慢病管理 / 体检筛查页
   - 血压、血糖、胆固醇、慢病复诊、health checks、预防管理
5. 女性健康 / 皮肤 / 特色服务页
   - 如果诊所 female GP 强，就做女性健康
   - 如果医生有 dermoscopy / skin check 优势，就做皮肤与小程序
   - 如果诊所线上复诊便利，也可替换为 telehealth / convenience page

### 方案 B，按品牌资产类型切
如果“5 文件”不是 5 个页面，而是 5 种素材类型，可用：
1. 行业总包
2. website-design
3. pdf-design
4. poster-design
5. social-design

但就这个行业而言，我仍然更建议先做方案 A。

## 我更推荐哪种
我更推荐 **方案 A**。

因为 GP 诊所的真实流量入口更像：
- 附近有没有中文医生
- 孩子发烧该先看哪里
- 想找会讲普通话 / 粤语的 GP
- 想做体检 / 疫苗 / 慢病复诊
- 想找女性健康或皮肤方向更合适的医生

也就是“具体问题驱动”，不是“抽象品牌驱动”。

## 当前资料对 5 文件的支撑度
目前已足够支撑：
- 综合首页
- 家庭全科 / 常见病页
- 儿童与家庭医疗页
- 慢病管理 / 体检页
- 女性健康页 或 皮肤 / telehealth 特色页

## 还可以继续补的内容
如果决定继续深化，我建议下一轮补 3 类：
1. **更具体的 FAQ 模版**
   - 比如首次就诊 FAQ、billing FAQ、儿童看诊 FAQ、telehealth FAQ
2. **更具体的社媒选题**
   - 比如“当地看病为什么先看 GP”“bulk billing 和 mixed billing 有什么区别”“孩子发烧先准备什么”
3. **更细的 suburb 本地化版本**
   - 比如 your area、your area、your area、your area、CBD 各自的华人社区表达差异

## 当前结论
- **可映射到 5 文件：是**
- **值得继续：是**
- **最值得优先继续的方向：家庭全科、儿童与家庭、慢病管理**
- **如果要快速落地获客页：先做综合首页 + 3 个核心业务页，再补一个特色服务页**
