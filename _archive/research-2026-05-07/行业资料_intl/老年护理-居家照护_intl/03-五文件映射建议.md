# 老年护理 / 居家照护五文件映射建议（AI 行动包格式）

更新时间：2026-04-10

## 已理解的既有结构
这个行业资料包的作用不是只写一个网站，而是为后续生成 **页面、销售说明、FAQ、社媒、顾问式咨询脚本、AI 协作流程** 提供统一底稿。

因此这里的“5 文件”不再按网站页面切，而是按 **AI 行动包** 的实际执行需要来切。

---

## 我对“是否适合做 AI 行动包”的判断
**非常适合，而且优先级高。**

原因：
1. 行业信息复杂，家庭经常听不懂，需要 AI 先做翻译与解释。
2. 咨询常常重复，适合沉淀 FAQ、分诊脚本、表单与流程。
3. 决策链长，涉及长者、子女、照护者三方，适合用 AI 做角色化内容。
4. 合规要求高，必须统一表达口径，AI 行动包很适合做“护栏”。
5. 很多咨询并不是立即成交，而是需要教育和跟进，AI 能提升前端承接效率。

---

## 推荐的 5 文件结构

### 1. Quick Start Guide
**作用**：给第一次接手这个行业的人，10 分钟内看懂怎么开始。

建议内容：
- 这个行业卖的到底是什么，不是什么
- 三类主要客户：长者、子女、主要照护者
- 两条主要路径：政府资助、私费启动
- 三类高频咨询：流程不懂、现在就急需、中文沟通需求
- 三个绝对不能踩的坑：夸大补助、夸大医疗能力、忽略家属决策
- 首次接待时必须先问的 8 个问题

适合产出：
- 新成员 onboarding
- 快速销售培训
- 首次咨询分流

---

### 2. Tool Guide
**作用**：告诉 AI 或团队，遇到不同场景时该调用什么工具、模板和判断规则。

建议内容：
- 咨询分诊规则
  - 政府资助咨询
  - 私费紧急咨询
  - 失智 / 高风险支持咨询
  - 出院后短期支持咨询
- 信息采集模板
  - 长者年龄、居住状态、语言、行动能力、记忆情况、现有支持、suburb、紧急程度
- 风险分级规则
  - 需要先找 provider 顾问
  - 需要先找 GP / hospital / emergency
- 文案调用规则
  - 对长者怎么说
  - 对子女怎么说
  - 对照护者怎么说
- 合规护栏
  - 不能承诺资格
  - 不能承诺费用结果
  - 不能假装自己是 My Aged Care 或政府部门
  - 涉及临床问题必须回到真实服务范围

适合产出：
- AI 客服规则
- 表单字段说明
- CRM 跟进标签逻辑
- 顾问脚本调用说明

---

### 3. Prompt Library
**作用**：沉淀高频可复用 prompt，让 AI 稳定产出咨询回复、页面、FAQ、跟进内容。

建议至少覆盖：
- 首次咨询回复 prompt
- 子女焦虑型咨询安抚 + 分流 prompt
- 长者不愿接受帮助时的话术 prompt
- 政府资助 vs 私费解释 prompt
- 失智家庭 FAQ 生成 prompt
- 服务介绍页生成 prompt
- suburb 本地化页面 prompt
- 即时通讯工具 / Instagram/TikTok科普贴 prompt
- 电话回访总结 prompt
- 顾问跟进短信 / 邮件 prompt

关键要求：
- 每个 prompt 都要内置合规提醒
- 默认输出不要夸大护理与补助承诺
- 默认同时照顾长者与家属两个视角

---

### 4. Workflow SOP
**作用**：把从流量进入到咨询、筛选、跟进、转化、回访的步骤写清楚。

建议内容：
- SOP 1：网站 / 广告线索进入后的 15 分钟内响应流程
- SOP 2：首次电话咨询流程
- SOP 3：判断是教育型线索还是紧急启动型线索
- SOP 4：需要政府资助时，如何解释 My Aged Care 路径
- SOP 5：需要私费立即开始时，如何推进评估与报价
- SOP 6：家属多人参与决策时，如何同步信息
- SOP 7：高风险情况的升级流程
- SOP 8：7 天内的 follow-up 节奏

关键要求：
- SOP 里要把“哪些话不能说”一起写进去
- 每个步骤都尽量可模板化，方便 AI 和人工协作

---

### 5. 7-Day Plan
**作用**：让团队在 7 天内把这个行业的最小可用 AI 行动包先跑起来。

建议节奏：
- Day 1：确认服务范围、区域、语言能力、私费 / 政府资助边界
- Day 2：整理 30 个高频问题，按长者 / 子女 / caregiver 分类
- Day 3：完成首次咨询问卷、分诊树、风险升级规则
- Day 4：完成首页文案、服务页文案、FAQ 初版
- Day 5：完成 10 条即时通讯工具 / Instagram/TikTok / 博客内容模板
- Day 6：完成 8 个客服 / 销售 prompt + 跟进模板
- Day 7：用真实或模拟线索跑一次，从咨询到 follow-up 全流程复盘

交付目标：
- 能接住首次咨询
- 能分清楚需求类型
- 能稳定说清政策与边界
- 能产出最基础的获客内容

## 整体评估

### 这个行业做 AI 行动包的价值
**很高。**

因为它本质上是一个：
- 高解释成本
- 高信任门槛
- 高重复咨询
- 高家庭协作
- 高合规要求

的行业。

### 最适合优先 AI 化的环节
1. 首次咨询分诊
2. FAQ 与政策解释
3. 家属沟通与跟进话术
4. 内容教育与社媒科普
5. suburb / 人群场景化页面生成

### 暂时不建议过度 AI 化的环节
1. 涉及复杂临床判断的建议
2. 对具体资格、补助金额、个人贡献的个案承诺
3. 高风险个案的最终服务决策
4. 投诉、事故、严重照护争议处理

### 当前结论
- **可映射到 5 文件：是**
- **最推荐的 5 文件形式：AI 行动包，而不是网站页面切法**
- **最值得优先落地的顺序：Quick Start Guide → Workflow SOP → Prompt Library → Tool Guide → 7-Day Plan**
- **如果要快速启动：先把首次咨询分诊、FAQ、家属沟通模板做出来，转化提升会最直接**
