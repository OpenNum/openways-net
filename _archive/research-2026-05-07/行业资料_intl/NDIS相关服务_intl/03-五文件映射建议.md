# NDIS相关服务后续五文件映射建议（AI 行动包格式）

更新时间：2026-04-10

## 已理解的既有结构
当前这套行业资料目录的作用，不是直接代替网站，而是先形成一个可执行行业底稿，供后续 AI 行动包、网站内容、海报、PDF、社媒、销售 SOP 统一调用。

对 NDIS 相关服务来说，后续最重要的不是“按页面切 5 份”，而是按 **运营动作、内容复用、合规控制、咨询转化** 来设计 5 个文件。

---

## 我对这个行业是否适合做 AI 行动包的判断
非常适合，而且比很多普通本地服务更需要。

原因：
1. 这是强解释型行业，FAQ 和流程价值很高
2. 咨询前置沟通多，适合用标准化 prompt 和 SOP 提效
3. 合规风险高，特别需要统一话术库
4. participant、家属、referrer、staff 之间经常需要多角色沟通模版
5. 不同团队成员如果各自乱写，很容易出现夸大 funding、价格、注册状态等问题

---

## 推荐的 5 文件结构

### 1. Quick Start Guide
**定位**：给新接手这个行业的人，30 分钟内先看懂“这个行业怎么说、先做什么、哪些不能乱说”。

**建议内容**：
- NDIS 行业一句话理解
- 目标客户是谁：participant、家属、转介人
- 优先主推哪些服务线：Support Coordination、In-home Support、Community Participation、Respite / STA、SIL
- 这个行业最重要的 5 个合规点
- 先问客户的 8 个问题
- 绝不能出现的 10 类高风险说法
- 推荐先产出的 3 类内容：FAQ、服务介绍、初步咨询分诊

**为什么重要**：
这个文件能先把团队拉到同一认知层，避免一上来就写成“普通家政”或“什么都能报”。

---

### 2. Tool Guide
**定位**：告诉执行人员在这个行业里，AI 应该帮什么，不应该帮什么。

**建议内容**：
- 可用工具场景
  - 双语 FAQ 起草
  - 服务页草稿
  - 家属沟通文案
  - suburb / language landing page 初稿
  - 电话咨询记录整理
  - staff matching intake notes 模版
- 慎用工具场景
  - funding eligibility 最终判断
  - 法律 / 注册结论
  - 个案 claim 承诺
  - clinical / restrictive / incident 相关敏感结论
- 输出前人工复核清单
  - 是否准确写明 registered / unregistered
  - 是否误导“都能 claim”
  - 是否收费表达透明
  - 是否把服务边界写清楚
  - 是否涉及 participant 隐私

**为什么重要**：
NDIS 是高风险行业，Tool Guide 的价值在于给 AI 设护栏，而不是只教“怎么生成更多内容”。

---

### 3. Prompt Library
**定位**：沉淀高频可复用 prompt，让团队稳定地产出双语、合规、转化导向内容。

**建议包含的 prompt 模块**：
- 写首页 hero + subheading，面向您所在的城市华人家庭
- 写 Support Coordination 服务介绍，不夸大 funding
- 写中文 FAQ，解释 registered vs unregistered provider
- 写社区参与服务页，强调 choice and control
- 写 Mandarin-speaking support worker 页面
- 写给家属的即时通讯工具短文 / Instagram/TikTok文案 / Facebook post
- 写初步咨询回复模版
- 写 service agreement 前置说明清单
- 写 complaint / feedback 页面文案
- 写转介合作介绍文案，面向 allied health / 社工 / 学校

**为什么重要**：
这个行业非常依赖“反复解释同一件事”，Prompt Library 能大幅减少重复劳动，同时保持口径一致。

---

### 4. Workflow SOP
**定位**：把“线索进来以后怎么走”标准化，而不是只停留在营销文案层面。

**建议流程**：
1. 线索进入
2. 收集基础信息
   - 服务区域
   - 语言需求
   - participant / 家属 / referrer 身份
   - 已有 plan 与否
   - management type
   - 当前痛点
3. 初步分诊
   - 更适合 coordination 还是 direct support
   - 是否需要 registered provider
   - 是否属于我们覆盖范围
4. 发送双语资料
   - 服务说明
   - 区域说明
   - 基本收费 / agreement / cancellation 说明
5. 安排 discovery call / intake
6. 内部匹配 staff 或服务路径
7. 发书面确认与 next steps
8. follow-up 与记录归档

**还应附带**：
- 不适配线索如何礼貌转介
- 高风险问题何时升级给负责人
- 哪些信息必须书面确认

**为什么重要**：
NDIS 咨询不是普通 quote 业务，真正影响转化的是 intake 清晰、沟通稳定、边界明确。

---

### 5. 7-Day Plan
**定位**：给团队一个一周内可落地的最小执行计划，快速把行业底稿变成能用的获客与交付资产。

**建议 7 天安排**：
- Day 1：统一行业认知，读 Quick Start Guide，确定主服务线
- Day 2：完成首页与 2 条核心服务线文案初稿
- Day 3：完成 FAQ、合规免责声明、注册状态说明模板
- Day 4：完成 Prompt Library 首版，覆盖咨询回复和社媒
- Day 5：完成 intake form、discovery call checklist、follow-up 模版
- Day 6：完成 2 个 suburb / language landing page 初稿
- Day 7：复核价格、claim、agreement、complaint 相关表述，并定版发布首批内容

**为什么重要**：
这个行业最容易卡在“信息太多，迟迟不发布”。7-Day Plan 的作用是让团队先做出一版可用且合规的最小资产集合。

---

## 整体评估

### 这个行业最适合 AI 行动包支持的环节
1. 双语解释与 FAQ
2. 初步需求分诊与 intake 文档
3. 服务页和区域页起草
4. 家属沟通与 follow-up 模版
5. 合规表达统一

### 这个行业不适合完全交给 AI 的环节
1. registration / eligibility 最终判断
2. 个案 funding 承诺
3. 复杂服务边界判断
4. 价格与 claim 争议处理
5. 投诉、incident、敏感个案决策

### 我对后续落地优先级的建议
最优先做：
- Quick Start Guide
- Prompt Library
- Workflow SOP

第二批做：
- Tool Guide
- 7-Day Plan

原因：
- 前三者直接影响团队是否会说、会接、会分诊
- 后两者更像管理和扩张工具，在有了初版服务资产后价值更大

---

## 当前一句话结论
NDIS 相关服务非常适合做成 AI 行动包，但必须按 **Quick Start Guide、Tool Guide、Prompt Library、Workflow SOP、7-Day Plan** 这 5 个执行文件来组织，而不是再按网站页面切。因为这个行业真正要解决的核心，不只是“写页面”，而是 **统一合规口径、提高解释效率、稳定 intake 流程、减少误导风险**。
