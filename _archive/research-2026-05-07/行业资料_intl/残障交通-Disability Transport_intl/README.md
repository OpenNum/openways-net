# 残障交通 Disability Transport

本目录整理了残障交通 / 无障碍出行 / 非急救医疗运输（NEMT）相关行业资料，供后续做市场判断、服务设计、AI接入方案、内容包装与客户沟通使用。

- `sources.md`：本次参考来源

## 这份资料的核心关注点
1. 服务流程是否标准化，能否被 AI 辅助或半自动化。
2. 预约、改期、派单、回单、对账这些重复劳动是否重。
3. 客户、家属、护理机构、医院、司机之间的信息传递是否容易断裂。
4. 合规和责任边界是否清楚，尤其是“运输服务”与“照护/医疗行为”的边界。
5. 哪些环节适合先做 AI：客服分流、预约收集、行前提醒、异常通知、回访总结、数据归档、内容生产。

## 适用范围
虽然起点可参考您所在的城市/当地市场，但结构按更通用的无障碍出行运营逻辑整理，也可迁移到美国 NEMT、英国 Dial-a-Ride / Community Transport、以及其他成熟城市的类似服务。

## 结论先看
- 这个行业不是单纯“开车接送”，而是“高沟通密度 + 高异常密度 + 高信任门槛”的服务业。
- 最重的成本往往不是油费，而是人工协调、爽约、临时改期、信息遗漏、资质维护、回款链路。
- AI 最适合先切进“信息整理与触达”层，不宜一开始就碰最终调度决策和风险承诺。
- 面向客户的价值主张，不应只讲“准时接送”，而应讲“让本人和家属更安心，且减少机构沟通成本”。
