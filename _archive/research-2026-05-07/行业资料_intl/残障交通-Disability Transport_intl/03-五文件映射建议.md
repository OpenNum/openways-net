# 03 五文件映射建议

> 目标：把“残障交通”行业研究，映射成一个可执行的 AI 行动包（AI Action Pack），方便后续直接落地为业务 SOP、知识库、客服助手、调度辅助系统或行业模板。

---

# AI 行动包总评估

## 行业适配度
**高。**

原因不是行业技术含量最高，而是它同时具备：
- 高频重复沟通
- 强规则、强例外
- 多角色协同
- 人工调度负担重
- 客户情绪敏感、信任门槛高
- 内容教育和流程解释非常重要

这类行业非常适合 AI 先做“信息中台 + 沟通自动化 + SOP 标准化”。

## AI 适合切入的优先级
### P1，最适合
- 预约 intake
- FAQ / 客服答疑
- 行前提醒
- 改期和取消收集
- 家属 / 机构状态通知
- 对话总结与记录归档
- 投诉/异常初步分类

### P2，适合做辅助
- 派车建议
- 客户分层与 recurring trip 识别
- 对账材料整理
- 内容生成与销售支持

### P3，不建议自动闭环
- 最终调度决策
- 风险责任判断
- 医疗建议
- 超边界承诺

## 整体建议
先从“前台沟通 + 中台记录”做起，再逐步接调度与运营系统。不要一开始把 AI 定义成“自动派车大脑”，更稳妥的定位是“预约与沟通协调助手”。

# 1. Quick Start Guide

## 适用对象
- 残障交通公司老板
- 调度主管
- 客服/预约团队
- support coordinator 合作方
- 想做行业 AI 方案的人

## 7 天内最小启动方式
### 第一步，先统一 intake
建立一个统一预约收集表，至少包含：
- 客户姓名与联系方式
- 家属/机构联系人
- 出发地、目的地、时间
- 单程/往返/等待
- 轮椅和辅助设备情况
- 陪同人数
- funding / payment 信息
- 特殊注意事项

### 第二步，建立 5 类标准消息模板
- 新客确认
- 行前提醒
- 改期/取消收集
- 异常通知
- 服务后回访

### 第三步，给每位客户生成“乘车档案”
目标不是存更多文字，而是减少重复询问。

### 第四步，只让 AI 做建议，不做最终承诺
AI 可以：收集、提醒、整理、生成摘要。
AI 不应：直接拍板高风险安排。

## Quick Win
如果只做一个功能，优先做：
**“预约信息自动收集 + 行前提醒 + 改期回收”**

这通常就能明显减少电话往返和 no-show。

# 2. Tool Guide

## 推荐工具层级

### A. 基础层，低成本马上能做
- 表单：Google Form / Typeform / Tally
- 表格数据库：Google Sheets / Airtable
- 沟通：短信、WhatsApp、邮件
- 自动化：Zapier / Make / n8n
- AI：ChatGPT / Claude / API-based assistant

**适合做什么**
- 收集预约
- 生成确认消息
- 自动归档客户信息
- 输出行程摘要

### B. 运营层，中期建设
- CRM / 客户档案系统
- 调度软件
- 司机 App / 司机打卡工具
- 票据和账单系统

**适合做什么**
- 司机与调度协同
- recurring trip 管理
- 异常追踪
- 对账与结算

### C. 增强层，高价值整合
- 地图与 ETA 服务
- 语音转文字
- 呼叫摘要
- BI 报表
- API 对接 broker / payer / booking 平台

## 工具使用原则
1. 先把字段标准化，再谈自动化。
2. 先打通消息链路，再打通高级调度。
3. 凡是涉及承诺和责任的动作，都保留人工确认。

# 3. Prompt Library

## Prompt 1，新客 intake 助手
你是一名残障交通预约助理。你的任务是收集一次预约所需的关键信息。请用友好、清楚、不过度承诺的语气，逐项确认以下内容：出发地、目的地、时间、是否单程或往返、是否需要等待、轮椅或辅助设备情况、是否有陪同者、是否有家属或机构联系人、付款或 funding 方式、是否有特殊注意事项。若需求超出运输与基础协助范围，请提醒需要人工确认适配性。

## Prompt 2，行前提醒生成器
请根据以下订单信息，生成一条简洁、礼貌、适合发给客户或家属的行前提醒消息。要包含：预约时间、接送地点、如需改动请尽快告知、如有轮椅/陪同信息变更请提前说明。语气要稳妥，不要作出绝对准时承诺。

## Prompt 3，异常通知助手
你是一名调度沟通助手。请根据以下异常情况，分别生成发给客户、家属、机构联系人三类对象的通知文案。要求：清楚说明当前情况、预计影响、下一步动作，并避免引发恐慌或过度承诺。

## Prompt 4，服务后总结
请把以下司机反馈、聊天记录和订单信息，整理成一份标准服务摘要。输出字段包括：订单编号、完成情况、是否准时、是否有异常、客户反馈、后续建议、是否适合标记为 recurring trip。

## Prompt 5，投诉分类与 SOP 建议
请将以下客户投诉归类为：迟到、沟通问题、适配问题、费用问题、司机服务问题、行程信息错误、其他。然后给出一版内部处理建议，包括：需要核对的事实、需联系的人、建议回复口径、是否需要升级处理。

## Prompt 6，机构合作页面文案
请为一家残障交通服务商生成面向 support coordinator / care team / 机构合作方的网页文案。重点强调：预约信息标准化、异常沟通、固定班次管理、减少电话往返、服务边界清晰。语气专业、可信，不要浮夸。

# 4. Workflow SOP

## SOP 1，新订单处理
1. 收到咨询
2. AI 收集标准字段
3. 检查是否缺少关键资料
4. 标记风险项，如轮椅不明、返程待定、联系人缺失
5. 人工确认可服务性
6. 创建订单并发送确认消息

## SOP 2，次日行程确认
1. 前一天自动发送确认提醒
2. 回收改期/取消信息
3. 标记未回复订单
4. 人工复核高风险订单
5. 向司机推送更新后的最终信息

## SOP 3，行程异常处理
1. 司机/系统触发异常
2. AI 生成客户、家属、机构通知草稿
3. 调度人工确认并发出
4. 更新订单状态
5. 事后自动生成异常摘要

## SOP 4，服务后归档
1. 司机提交完成状态
2. AI 汇总服务记录
3. 判断是否加入 recurring trip 候选
4. 推送回访消息
5. 归档供后续对账或投诉处理

## SOP 5，机构合作客户维护
1. 汇总高频订单和异常率
2. 每周输出简报
3. 标记需优化的预约字段
4. 对 recurring 路线提出标准化建议

# 5. 7-Day Plan

## Day 1
梳理当前预约流程，列出所有输入渠道，电话、即时通讯工具/WhatsApp、短信、表单、邮件。

## Day 2
确定统一字段标准，完成一版 intake 表单和客户档案模板。

## Day 3
制作 5 个标准消息模板，新客确认、行前提醒、改期取消、异常通知、服务后回访。

## Day 4
把 AI 接到 intake 和消息生成环节，先做人工审核版。

## Day 5
选择 10 到 20 个真实订单试运行，记录缺字段、误判、遗漏点。

## Day 6
优化 SOP，补上高风险规则，例如轮椅类型不明、返程待定、医院延误、联系人缺失。

## Day 7
复盘结果，统计节省的人工时间、减少的来回沟通、客户反馈，并决定是否进入第二阶段，接司机端或调度系统。

# 最终判断
残障交通行业非常适合做 AI 行动包，尤其适合围绕以下主题展开：
- 预约标准化
- 调度辅助
- 家属与机构沟通
- 异常管理
- 内容获客
- 档案与回单沉淀
